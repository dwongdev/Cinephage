## Why a Single Application?

Unlike traditional \*arr applications that run as separate services, Cinephage unifies everything into one application.

## Why YAML for Indexers?

All indexer definitions use YAML instead of being hardcoded.

## Why SQLite?

Cinephage uses SQLite instead of PostgreSQL or MySQL.
